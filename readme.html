ODIN Documentation

Intro

This is just a place to put some notes for the project. Hopefully the presence of this document steers future programmers from having a bad time.

Templates

Templates are to be put in the '__library/__template/' directory and be named anything that adheres to the following convention. This allows them to easily be parsed by humans.

[name]Template.html

Templates are rendered using Mustache.js. Templates in Odin should be accessed using the TEMPLATES module which ensures that they are cached and not grabbed multiple times.

Templates can be used in conjunction with controllers.

Controllers

Contollers, like templates, are kept in the '__library/__template/' directory. They should be named anything that adheres to the follwing convention.

[name]Controller.js

Furthermore, not anything can be a controller. The controller code is just an extension onto a controller object. A controller class should follow this convention:

function [name](){ // object is the object that will be passed to the controller and then passed to the template. this.beforeRender = function(object){ //...code... } }
